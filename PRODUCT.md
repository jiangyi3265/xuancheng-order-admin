# 玄成接单系统：产品语境

## 产品是什么

面向玄程科技内部两人团队的接单与交付工作台。它把客户需求、项目状态、沟通记录、回款和 AI 改码任务放在同一套后台，减少在聊天工具、表格和代码仓库之间来回切换。

## 核心用户

- 总裁：接单、分派、验收、回款、批准 AI 候选代码。
- 副总裁：开发推进、问题处理、提交验收、查看 AI 执行结果。

两类用户都高频使用桌面端，也需要在手机上完成查看和简单操作。用户熟悉订单业务，但不应被要求理解 Agent、围栏令牌等底层术语。

## 核心任务

1. 快速知道今天需要自己处理什么。
2. 查看一个项目的需求、沟通、Bug 和交付状态。
3. 给业务项目绑定一次代码仓库和交付策略，之后客户 Bug 自动进入 Worker。
4. 正常任务自动改码、验证、合并、部署并通知客户，人工只处理异常和高风险任务。
5. 任何时候都能看见当前执行机、PR、部署记录、客户交付结果和失败原因。

## AI 开发边界

- 每个项目可选择人工、自动产出 PR、全自动交付三种模式。
- 全自动交付仅在策略检查和确定性验证通过后合并，并等待指定 GitHub Actions 部署成功才通知客户。
- 控制面在云端；高耗时改码、构建和测试在自有机器的一次性容器内执行。
- GitHub 托管 Actions 分钟只用于主干部署和少量关键 CI。
- 代码状态、测试证据和审核动作必须可追溯。
- 自动执行失败最多重试一次，仍失败则进入异常中心并转人工，不向客户交付失败版本。

## 产品气质

清晰、克制、可靠、偏高密度。它是内部生产工具，不是营销页面，也不是带霓虹效果的“AI 驾驶舱”。重要状态要醒目，但不能让整页都处于警报状态。
