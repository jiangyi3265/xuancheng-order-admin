# 玄成接单系统：界面约束

## 视觉基线

- 延续 Element Plus 和现有后台结构，不另造组件体系。
- 字体使用系统无衬线字体；正文 13–14px，页面标题 20px。
- 页面背景 `#f0f2f5`，内容面板白色，主文字 `#303133`，次文字 `#909399`。
- 主色沿用 Element Plus 蓝色；红色只表示失败、逾期和破坏性操作。
- 圆角以 6–8px 为主，边框优先于大阴影。

## 信息层级

- 页面顶部只保留标题、说明和主要动作。
- 状态汇总使用单条统计带，不拆成多张装饰卡片。
- 主任务列表承担页面主体；详情、日志和配置进入抽屉或对话框。
- 代码指纹、分支名使用等宽字体，并允许复制。

## 交互

- 任务状态必须同时使用文字，不依赖颜色表达。
- 高风险与批准动作必须二次确认。
- 加载、空状态、失败原因和不可操作原因要明确。
- 移动端优先保留查看、刷新和关键审核；复杂配置允许横向滚动或进入弹窗。
- 不使用渐变文字、玻璃拟态、悬浮胶囊导航、装饰性大标题或无意义动画。
